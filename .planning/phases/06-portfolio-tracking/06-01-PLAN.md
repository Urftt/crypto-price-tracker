# Plan 06-01: Backend — Storage Layer, Aggregation Service, and Export

**Phase:** 6 — Portfolio Tracking
**Depends on:** Nothing (first plan in phase)
**Estimated tasks:** 5

## Goal

Build the complete backend for portfolio tracking: the `Holding` dataclass, SQLite storage layer with CRUD operations, portfolio aggregation service with P&L calculations, CSV/JSON export, and comprehensive tests for all of it. This plan produces the foundation that Plan 06-02 wires into the CLI and web surfaces.

## Context

**Existing patterns to follow:**
- `@dataclass(slots=True)` in `models.py` (see `CoinData`)
- `get_top_coins(top_n=100)` in `api.py` returns `list[CoinData]` — portfolio aggregation needs this for current prices
- Tests use pytest with `uv run pytest tests/ -x -q`
- Console injection pattern for testable Rich output

**Key architectural decisions (from RESEARCH.md):**
- stdlib `sqlite3` — no ORM, no new dependencies
- WAL journal mode on every connection for CLI/web concurrency
- Connection-per-call pattern (open, use `with conn:`, close)
- `CREATE TABLE IF NOT EXISTS` on every `get_connection()` call (idempotent schema)
- Manual XDG path resolution (3 lines, no `platformdirs`)
- Store dates as ISO 8601 TEXT (`YYYY-MM-DD`)
- `?` parameter binding for all SQL (never f-strings)
- Prices come from API, NOT stored in portfolio DB

### Task 2: Create `portfolio_db.py` — SQLite storage layer
**File:** `src/crypto_price_tracker/portfolio_db.py`
**Action:** create

This module handles all SQLite interactions. Every public function accepts an optional `db_path: Path | None` parameter (defaults to XDG path) so tests can pass a temp file path.

**Imports:**
```python
from __future__ import annotations

import sqlite3
from datetime import date
from pathlib import Path

from crypto_price_tracker.models import Holding
```

**Functions to implement (in this order):**

1. `_get_default_db_path() -> Path` — Resolve `$XDG_DATA_HOME/crypto-tracker/portfolio.db`, defaulting to `~/.local/share/crypto-tracker/portfolio.db`. Create parent dirs with `mkdir(parents=True, exist_ok=True)`. Use `os.environ.get("XDG_DATA_HOME")` with fallback to `Path.home() / ".local" / "share"`.

2. `_holding_factory(cursor: sqlite3.Cursor, row: tuple) -> Holding` — Custom row factory: extract column names from `cursor.description`, zip with row values, construct `Holding(**dict(zip(names, row)))`.

3. `get_connection(db_path: Path | None = None) -> sqlite3.Connection` — Open connection to `db_path or _get_default_db_path()`. Set `PRAGMA journal_mode=WAL`, set `PRAGMA foreign_keys=ON`, assign `conn.row_factory = _holding_factory`, call `_ensure_schema(conn)`, return conn.

4. `_ensure_schema(conn: sqlite3.Connection) -> None` — Execute `CREATE TABLE IF NOT EXISTS holdings (id INTEGER PRIMARY KEY AUTOINCREMENT, symbol TEXT NOT NULL, amount REAL NOT NULL CHECK(amount > 0), buy_price REAL NOT NULL CHECK(buy_price > 0), buy_date TEXT NOT NULL)`.

5. `add_holding(symbol: str, amount: float, buy_price: float, buy_date: str | None = None, *, db_path: Path | None = None) -> int` — Default `buy_date` to `date.today().isoformat()` if None. Validate date with `date.fromisoformat(buy_date)` (raises ValueError on bad format). Upper-case symbol. Open connection, `INSERT INTO holdings ... VALUES (?, ?, ?, ?)` inside `with conn:`, return `cursor.lastrowid`, close conn.

6. `remove_holding(holding_id: int, *, db_path: Path | None = None) -> bool` — `DELETE FROM holdings WHERE id = ?`. Return `cursor.rowcount > 0`. Close conn.

7. `update_holding(holding_id: int, *, amount: float | None = None, buy_price: float | None = None, buy_date: str | None = None, db_path: Path | None = None) -> bool` — Build SET clause dynamically from non-None kwargs. If no kwargs provided, return False. Validate buy_date if provided. Execute UPDATE, return `cursor.rowcount > 0`. Close conn.

8. `get_all_holdings(*, db_path: Path | None = None) -> list[Holding]` — `SELECT * FROM holdings ORDER BY symbol, buy_date`. Return `cursor.fetchall()`. Close conn.

9. `get_holdings_by_symbol(symbol: str, *, db_path: Path | None = None) -> list[Holding]` — `SELECT * FROM holdings WHERE symbol = ? ORDER BY buy_date` with `symbol.upper()`. Return `cursor.fetchall()`. Close conn.

**Important:** Every function that opens a connection must close it after use. Use `try/finally` to guarantee cleanup: `conn = get_connection(...); try: ... return result; finally: conn.close()`. Never store connections as module-level state.

**Verify:** `python -c "from crypto_price_tracker.portfolio_db import get_connection; import tempfile, pathlib; p = pathlib.Path(tempfile.mktemp(suffix='.db')); c = get_connection(p); print('OK:', p); c.close()"`

### Task 3: Create `portfolio.py` — Aggregation service and export
**File:** `src/crypto_price_tracker/portfolio.py`
**Action:** create

This module computes aggregated portfolio views and handles export. It does NOT touch the database directly — it takes `list[Holding]` as input.

**Imports:**
```python
from __future__ import annotations

import csv
import io
import json
from collections import defaultdict
from dataclasses import asdict, dataclass

from crypto_price_tracker.models import CoinData, Holding
```

**Dataclasses to define:**

```python
@dataclass(slots=True)
class PortfolioRow:
    """Aggregated view of one coin across all lots."""
    symbol: str
    total_amount: float
    avg_buy_price: float
    current_price: float | None   # None = coin not in top 100
    total_cost: float
    current_value: float | None   # None = coin not in top 100
    pnl_eur: float | None         # None = coin not in top 100
    pnl_pct: float | None         # None = coin not in top 100
    allocation_pct: float | None  # None = coin not in top 100
    change_24h: float | None      # None = coin not in top 100

@dataclass(slots=True)
class PortfolioSummary:
    """Complete portfolio aggregation result."""
    rows: list[PortfolioRow]
    total_value: float     # Sum of current_value for priced coins + cost basis for unpriced
    total_cost: float      # Sum of all cost bases
    total_pnl_eur: float   # total_value - total_cost
    total_pnl_pct: float   # (total_pnl_eur / total_cost) * 100, or 0 if no cost
```

**Functions to implement:**

1. `aggregate_portfolio(holdings: list[Holding], prices: dict[str, CoinData]) -> PortfolioSummary` —
   - Group holdings by symbol using `defaultdict(list)` (imported from `collections`).
   - For each symbol group, compute: `total_amount = sum(h.amount)`, `total_cost = sum(h.amount * h.buy_price)`, `avg_buy_price = total_cost / total_amount`.
   - If symbol is in `prices` dict: compute `current_value = total_amount * prices[symbol].price`, `pnl_eur = current_value - total_cost`, `pnl_pct = (pnl_eur / total_cost) * 100` (guard div-by-zero), `change_24h = prices[symbol].change_24h`.
   - If symbol NOT in `prices` dict: set `current_price`, `current_value`, `pnl_eur`, `pnl_pct`, `allocation_pct`, `change_24h` all to `None`. This implements the "N/A" behavior for coins not in top 100.
   - Compute `total_value` as: sum of `current_value` for priced coins + sum of `total_cost` for unpriced coins (included at cost basis per locked decision).
   - Compute `allocation_pct` for each row: `(current_value / total_value) * 100` for priced coins; `None` for unpriced.
   - Sort rows by `current_value` descending (priced coins first, then unpriced by `total_cost` descending).
   - Round all float display values to 2 decimal places.
   - Return `PortfolioSummary(rows=rows, total_value=..., total_cost=..., total_pnl_eur=..., total_pnl_pct=...)`.

2. `export_csv(holdings: list[Holding]) -> str` —
   - Use `csv.DictWriter` with `io.StringIO()`.
   - Fieldnames: `["id", "symbol", "amount", "buy_price", "buy_date"]`.
   - Write header + one row per holding using `asdict(h)`.
   - Return the string.

3. `export_json(holdings: list[Holding]) -> str` —
   - Return `json.dumps([asdict(h) for h in holdings], indent=2)`.

**Verify:** `python -c "from crypto_price_tracker.portfolio import PortfolioRow, PortfolioSummary, export_csv, export_json; print('imports OK')"`
