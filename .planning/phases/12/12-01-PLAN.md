### Step 2: Create report.py — HTML report generation and PDF conversion
**File:** `src/crypto_price_tracker/report.py`
**Action:** create
**Why:** Central module for generating rich PDF reports and notification summaries from portfolio, price, watchlist, and alert data.

Create `src/crypto_price_tracker/report.py` with these functions:

**`generate_report_html(portfolio, coins, watchlist, alerts) -> str`**
- Takes `PortfolioSummary`, `list[CoinData]`, `list[WatchlistEntry]`, `list[PriceAlert]`
- Builds a complete HTML document with inline CSS (xhtml2pdf-compatible — CSS 2.1 only, no flexbox/grid)
- Sections: header with "Crypto Portfolio Report" + timestamp, Portfolio Summary (total value, total P&L), Holdings table (symbol, amount, avg buy, value, P&L, allocation), Top Prices table (rank, symbol, name, price EUR, 24h%), Watchlist table (symbol, tags), Active Alerts table (symbol, direction, target price)
- Green (#3fb950) for positive P&L/change, red (#f85149) for negative — matches project theme
- Dark header row (#0d1117 background, #e6edf3 text) on all tables
- EUR formatting: `f"EUR {value:,.2f}"` matching project convention
- Use simple HTML tables with inline styles. Stick to: background-color, color, font-size, font-weight, border, padding, text-align, width, border-collapse
- Skip sections with empty data (e.g. no watchlist entries = no watchlist section)

**`html_to_pdf(html: str) -> bytes`**
- Uses `io.BytesIO` buffer and `xhtml2pdf.pisa.CreatePDF(html, dest=buffer)`
- Raises `RuntimeError` if `pisa_status.err` is non-zero
- Returns `buffer.getvalue()` (NOT buffer directly — avoids BytesIO position bug)

**`build_summary_text(portfolio: PortfolioSummary) -> str`**
- Plain text summary for Telegram notifications
- Format with `<pre>` block for fixed-width alignment (Telegram does not render HTML tables)
- Include: total portfolio value, total P&L (EUR + %), top 5 holdings by value with their P&L
- Keep under 4096 characters (Telegram message limit) — truncate with "... and N more" if needed
- Use `parse_mode="HTML"` compatible tags only: `<b>`, `<i>`, `<code>`, `<pre>`

**`build_summary_html(portfolio: PortfolioSummary) -> str`**
- HTML summary for email notifications
- Simple HTML table with same green/red P&L coloring
- Include: portfolio summary header, holdings table, total footer

Import from existing modules:


### Step 3: Create notify.py — Telegram and email notification channels
**File:** `src/crypto_price_tracker/notify.py`
**Action:** create
**Why:** Encapsulates notification delivery to Telegram Bot API and SMTP email, with auto-detection of configured channels via environment variables.

Create `src/crypto_price_tracker/notify.py` with these functions:

**`send_telegram(message: str) -> bool`**
- Read `CRYPTO_TELEGRAM_TOKEN` and `CRYPTO_TELEGRAM_CHAT_ID` from `os.environ`
- If either is missing, return `False` (channel not configured)
- POST to `https://api.telegram.org/bot{token}/sendMessage` using `httpx.post()` (httpx is already a project dependency)
- JSON body: `{"chat_id": chat_id, "text": message, "parse_mode": "HTML"}`
- Call `response.raise_for_status()` — let HTTP errors propagate to caller
- Return `True` on success

**`send_email(subject: str, html_body: str, text_body: str) -> bool`**
- Read env vars: `CRYPTO_SMTP_HOST`, `CRYPTO_SMTP_PORT` (default "587"), `CRYPTO_SMTP_USER`, `CRYPTO_SMTP_PASS`, `CRYPTO_SMTP_FROM`, `CRYPTO_SMTP_TO`
- If `CRYPTO_SMTP_HOST` is missing, return `False` (channel not configured)
- Build `MIMEMultipart("alternative")` — attach plain text FIRST, then HTML (email clients render last compatible part)
- Use `ssl.create_default_context()` for TLS
- Connect with `smtplib.SMTP(host, port)`, call `server.starttls(context=context)`
- Login with user/password if both are set
- Call `server.sendmail(from_addr, to_addr, msg.as_string())`
- Return `True` on success

**`send_summary(text_message: str, html_message: str) -> list[str]`**
- Try `send_telegram(text_message)` — catch and print errors but continue to next channel
- Try `send_email("Crypto Portfolio Summary", html_message, text_message)` — catch and print errors
- Return list of channel names that succeeded (e.g. `["telegram", "email"]`)

Use only stdlib imports plus httpx (existing dep):
```python
import os, smtplib, ssl
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
import httpx
```

**File:** `src/crypto_price_tracker/cli.py`
**Action:** modify
**Why:** Wire the new report and notify modules into the CLI via `crypto export` and `crypto summary send` subcommands.

Note: `get_all_alerts` is already imported. `get_all_watchlist_entries` is already imported. Only add `report` and `notify` imports.

**Add command functions** (after `cmd_watchlist_tag`, before `cmd_chart`):

`cmd_export(args)`:
- Fetch live prices: `coins, source = get_top_coins_with_fallback(top_n=100)`
- Fetch portfolio: `holdings = get_all_holdings()`, `prices = {c.symbol: c for c in coins}`, `portfolio = aggregate_portfolio(holdings, prices)`
- Fetch watchlist: `watchlist = get_all_watchlist_entries()`
- Fetch alerts: `alerts = get_all_alerts()`
- Generate HTML: `html = generate_report_html(portfolio, coins, watchlist, alerts)`
- Convert to PDF: `pdf_bytes = html_to_pdf(html)`
- Determine output filename: use `args.output` if provided, else auto-name as `crypto-report-{YYYY-MM-DD}.pdf`
- Write PDF bytes to file
- Print: `f"Report saved to {output_path}"`

`cmd_summary_send(args)`:
- Fetch live prices and aggregate portfolio (same pattern as cmd_export)
- Build text: `text = build_summary_text(portfolio)`
- Build HTML: `html = build_summary_html(portfolio)`
- Send: `channels = send_summary(text, html)`
- If channels: print `f"Summary sent to: {', '.join(channels)}"`
- If no channels: print `"No notification channels configured. Set CRYPTO_TELEGRAM_TOKEN/CRYPTO_SMTP_HOST environment variables."`

**Add argument parsers** in `main()` (after the watchlist subcommand block, before chart_parser):
