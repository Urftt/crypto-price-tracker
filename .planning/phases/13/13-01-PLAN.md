### Step 2: Create PWA icons in frontend/public/
**File:** `frontend/public/` (new directory)
**Action:** create
**Why:** PWA requires icon files at specific sizes for installation and splash screens. Icons are served from the `public/` directory which Vite copies verbatim to the build output.

Create `frontend/public/` directory and add four icon files:

**`frontend/public/favicon.svg`** — SVG source icon (crypto-themed, simple Bitcoin-inspired design using project accent color #58a6ff):
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512">
  <rect width="512" height="512" rx="64" fill="#0d1117"/>
  <circle cx="256" cy="256" r="180" fill="none" stroke="#58a6ff" stroke-width="24"/>
  <text x="256" y="300" font-family="monospace" font-size="220" font-weight="bold" fill="#58a6ff" text-anchor="middle">$</text>
</svg>
```

Then generate PNG icons from the SVG. Use a simple approach: create minimal PNG placeholder icons using a Python script that writes valid PNG files:

```bash
cd /home/james-turing/repos/crypto-price-tracker-v2
python3 -c "
import struct, zlib

def create_png(width, height, r, g, b):
    \"\"\"Create a minimal solid-color PNG.\"\"\"
    def chunk(ctype, data):
        c = ctype + data
        return struct.pack('>I', len(data)) + c + struct.pack('>I', zlib.crc32(c) & 0xffffffff)

    raw = b''
    for _ in range(height):
        raw += b'\x00' + bytes([r, g, b]) * width

    return (b'\x89PNG\r\n\x1a\n' +
            chunk(b'IHDR', struct.pack('>IIBBBBB', width, height, 8, 2, 0, 0, 0)) +
            chunk(b'IDAT', zlib.compress(raw)) +
            chunk(b'IEND', b''))

# Project background color #0d1117 = rgb(13, 17, 23)
for name, size in [('pwa-192x192.png', 192), ('pwa-512x512.png', 512), ('apple-touch-icon-180x180.png', 180)]:
    with open(f'frontend/public/{name}', 'wb') as f:
        f.write(create_png(size, size, 13, 17, 23))
    print(f'Created {name} ({size}x{size})')
"
```

Also create `frontend/public/favicon.ico` by copying the 192px PNG (browsers accept PNG-in-ICO):
```bash
cp frontend/public/pwa-192x192.png frontend/public/favicon.ico
```

Files created in `frontend/public/`:

- `favicon.ico` — browser tab icon
- `pwa-192x192.png` — Android home screen
- `pwa-512x512.png` — Android splash / install dialog

### Step 3: Configure vite-plugin-pwa in vite.config.js
**File:** `frontend/vite.config.js`
**Action:** modify
**Why:** Add the VitePWA plugin with manifest fields, service worker caching strategies, and SSE exclusion.

Replace the entire `vite.config.js` with:

```js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
    VitePWA({
      registerType: 'autoUpdate',
      manifest: {
        name: 'Crypto Price Tracker',
        short_name: 'CryptoTracker',
        description: 'Real-time crypto prices, portfolio tracking, and alerts',
        start_url: '/',
        display: 'standalone',
        theme_color: '#0d1117',
        background_color: '#0d1117',
        icons: [
          { src: 'pwa-192x192.png', sizes: '192x192', type: 'image/png' },
          { src: 'pwa-512x512.png', sizes: '512x512', type: 'image/png' },
          { src: 'pwa-512x512.png', sizes: '512x512', type: 'image/png', purpose: 'maskable' },
        ],
      },
      manifestFilename: 'manifest.json',
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
        navigateFallback: '/index.html',
        navigateFallbackDenylist: [/^\/api\//],
        runtimeCaching: [
          {
            urlPattern: /\/api\/prices\/stream/,
            handler: 'NetworkOnly',
          },
          {
            urlPattern: /^\/api\//,
            handler: 'NetworkFirst',
            options: {
              cacheName: 'api-cache',
              expiration: { maxEntries: 50, maxAgeSeconds: 3600 },
              cacheableResponse: { statuses: [0, 200] },
            },
          },
        ],
      },
    }),
  ],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
  build: {
    outDir: '../src/crypto_price_tracker/static',
    emptyOutDir: true,
  },
});
```

Key configuration decisions:
- **`registerType: 'autoUpdate'`** — silently updates the service worker when new content deploys; no update prompt needed for a dashboard app
- **`manifestFilename: 'manifest.json'`** — outputs `manifest.json` instead of the default `manifest.webmanifest` because Python's `mimetypes` module does not recognize `.webmanifest`, causing FastAPI's `FileResponse` to serve it as `application/octet-stream`. Chrome rejects manifests with wrong MIME type. Using `.json` ensures `application/json` which Chrome accepts.
- **`navigateFallback: '/index.html'`** — explicitly set (matches generateSW default) for SPA routing clarity
- **`navigateFallbackDenylist: [/^\/api\//]`** — prevents the service worker from returning `index.html` for API navigation requests
- **SSE `NetworkOnly` rule** — explicit `handler: 'NetworkOnly'` for `/api/prices/stream` ensures the service worker never caches or interferes with the SSE connection. This is ordered BEFORE the general `/api/` NetworkFirst rule so it matches first. Without this, SSE requests would pass through the SW fetch handler with no matching route, which is fragile.
- **API `NetworkFirst` rule** — caches all other API GET responses for offline viewing (portfolio, alerts, watchlist, coin data)
- **`globPatterns`** — precaches all static assets (JS, CSS, HTML, icons) so the app shell loads offline
- **Manifest `theme_color` and `background_color`** both use `#0d1117` to match the project's GitHub-dark theme

### Step 4: Update index.html with PWA meta tags
**File:** `frontend/index.html`
**Action:** modify
**Why:** Add meta tags required for PWA installability and iOS support. The manifest link is auto-injected by vite-plugin-pwa, but meta tags must be added manually.

Replace the `<head>` section of `frontend/index.html`:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Crypto Prices -- EUR</title>
    <meta name="description" content="Real-time crypto prices, portfolio tracking, and alerts">
    <meta name="theme-color" content="#0d1117">
    <link rel="icon" href="/favicon.ico" sizes="any">
    <link rel="icon" href="/favicon.svg" type="image/svg+xml">
    <link rel="apple-touch-icon" href="/apple-touch-icon-180x180.png">
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>
```

Added tags:
- `<meta name="description">` — required for Lighthouse PWA audit
- `<meta name="theme-color">` — sets browser chrome color on mobile (matches #0d1117 bg)
- `<link rel="icon">` — favicon for browser tabs (both ICO and SVG for modern browsers)
- `<link rel="apple-touch-icon">` — iOS home screen icon (180x180)

Note: The `<link rel="manifest">` tag is auto-injected by vite-plugin-pwa during build — do NOT add it manually.
