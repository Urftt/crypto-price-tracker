### Step 2: Add null guard for `allocation_pct` in PortfolioTable
**File:** `frontend/src/components/PortfolioTable.jsx`
**Action:** modify
**Why:** `allocation_pct` can be `null` for unpriced holdings (coins outside top 100) or when total_value is 0. Calling `.toFixed()` on `null` throws a TypeError that crashes the entire portfolio table.

Change line 84 from:
```jsx
<td className="py-1.5 px-3 text-right text-sm">{row.allocation_pct.toFixed(1)}%</td>
```
To:
```jsx
<td className="py-1.5 px-3 text-right text-sm">{row.allocation_pct != null ? `${row.allocation_pct.toFixed(1)}%` : 'N/A'}</td>
```

Also add null guards for `current_price`, `current_value`, `pnl_eur`, and `pnl_pct` which can all be `null` for unpriced holdings:
- Line 80: `{row.current_price != null ? formatEUR(row.current_price) : 'N/A'}`
- Line 81: `{row.current_value != null ? formatEUR(row.current_value) : 'N/A'}`
- Line 82: P&L EUR — use `row.pnl_eur != null` check
- Line 83: P&L % — use `row.pnl_pct != null` check

### Step 5: Fix BitvavoClient.get_top_coins() protocol signature
**File:** `src/crypto_price_tracker/api.py`
**Action:** modify
**Why:** The `ExchangeClient` protocol defines `get_top_coins(self, top_n: int = 20)` but `BitvavoClient.get_top_coins()` takes no arguments. This is a structural mismatch that static type checkers would flag.
