**Step 4: Create `frontend/src/components/ui/Button.jsx`**

Export a named `Button` component with these exact props: `variant` (default: `'primary'`), `size` (default: `'md'`), `loading` (default: `false`), `className` (default: `''`), `children`, `disabled`, plus `...props` spread.

Variant class map (exact classes from codebase audit):
- `primary`: `'bg-accent text-bg font-bold hover:bg-accent/80'`
- `secondary`: `'bg-card border border-border text-text hover:border-border-light'`
- `danger`: `'text-down hover:text-down/80'`
- `ghost`: `'text-text-muted hover:text-text'`

Size class map:
- `xs`: `'px-1 py-0.5 text-xs'`
- `sm`: `'px-2.5 py-0.5 text-xs'`
- `md`: `'px-4 py-1.5 text-sm'`

Base classes: `'rounded cursor-pointer transition-colors disabled:opacity-50 disabled:cursor-not-allowed'`

CRITICAL: Do NOT set a default `type` attribute. Let it pass through via `...props`. This is because `<button>` defaults to `type="submit"` in HTML, and many existing buttons rely on this. Non-submit buttons in the codebase already explicitly set `type="button"`.

When `loading` is true: set `disabled` to true and render `'Submitting...'` instead of `children`.

Concatenate: `${base} ${variants[variant]} ${sizes[size]} ${className}`.

**Step 5: Create `frontend/src/components/ui/Input.jsx`**

Export a named `Input` component with these exact props: `label`, `error`, `className` (default: `''`), `id`, plus `...props` spread.

Generate `inputId` from `id || label?.toLowerCase().replace(/\s+/g, '-')`.

Render structure:
1. Outer `<div className={className}>` wrapper
2. If `label` is truthy: `<label htmlFor={inputId} className="text-text-muted text-xs mb-0.5 block">{label}</label>`
3. `<input id={inputId} className="bg-bg border rounded px-3 py-1.5 text-text text-sm focus:border-accent focus:outline-none w-full ${error ? 'border-down' : 'border-border'}" {...props} />`
4. If `error` is truthy: `<p className="text-down text-xs mt-0.5">{error}</p>`

NOTE: The `w-full` class is on the input; width control happens via `className` on the outer div. This differs from existing inline patterns where `w-24` etc. was on the input itself. Callers will use `className="w-24"` on Input and the inner input fills its container.

If no `label` is provided but an `aria-label` prop exists, it passes through via `...props` automatically.

**Step 6: Create `frontend/src/components/ui/Table.jsx`**

Export three named components: `Table`, `Th`, `Td`.

`Table`: Render `<table className={\`w-full ${className || ''}\`} {...props} />`.

`Th`: Props: `align` (default: `'left'`), `className` (default: `''`), plus `...props`.
CRITICAL: Use an explicit `alignMap` object for Tailwind alignment. Do NOT use dynamic `text-${align}` construction -- Tailwind's JIT scanner cannot detect dynamic class names.

```js
const alignMap = { left: 'text-left', right: 'text-right', center: 'text-center' };
```

Render: `<th className={\`text-text-muted text-xs uppercase py-1.5 px-3 border-b border-border ${alignMap[align]} ${className}\`} {...props} />`

`Td`: Same `align` prop with same `alignMap`. Render: `<td className={\`py-1.5 px-3 ${alignMap[align]} ${className}\`} {...props} />`

**Step 7: Create `frontend/src/components/ui/Modal.jsx`**

Export a named `Modal` component with props: `open`, `onClose`, `children`.

Implementation:
1. Use `useRef` for `modalRef` on the content div.
2. Escape key handler: `useEffect` that adds a `keydown` listener when `open` is true. On `Escape` key, call `onClose()`. Clean up on unmount / when `open` changes.
3. Focus trap: Second `useEffect` when `open` is true. Query focusable elements: `'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'`. Focus the first element on open. Add `keydown` listener for Tab key: if shift+Tab on first element, wrap to last; if Tab on last element, wrap to first. IMPORTANT: Re-query `focusable` elements inside the Tab handler (not just on mount) to handle React rerenders.
4. If `!open`, return `null`.
5. Render backdrop: `<div className="fixed inset-0 bg-black/70 z-40 flex items-center justify-center" onClick={onClose}>`
6. Render content: `<div ref={modalRef} className="bg-card border border-border-light rounded-lg p-6 max-w-xl w-full mx-4 relative z-50" onClick={(e) => e.stopPropagation()}>{children}</div>`

These classes match the existing CoinModal backdrop and content div exactly.

**Step 8: Create `frontend/src/components/ui/Badge.jsx`**

Export a named `Badge` component with props: `colorScheme`, `variant`, `className` (default: `''`), `children`, plus `...props`.

Color scheme map (for tag badges):
```js
const colorSchemes = {
  blue: 'bg-blue-900/50 text-blue-300 border-blue-700',
  purple: 'bg-purple-900/50 text-purple-300 border-purple-700',
  green: 'bg-green-900/50 text-green-300 border-green-700',
  yellow: 'bg-yellow-900/50 text-yellow-300 border-yellow-700',
  orange: 'bg-orange-900/50 text-orange-300 border-orange-700',
  red: 'bg-red-900/50 text-red-300 border-red-700',
  gray: 'bg-border text-text-muted border-border',
};
```

Variant map (for status badges):
```js
const variants = {
  up: 'text-up bg-up/10',
  down: 'text-down bg-down/10',
};
```

Logic: If `colorScheme` is provided, use `colorSchemes[colorScheme]` and include `border` in base classes. If `variant` is provided, use `variants[variant]`. Default fallback: `colorSchemes.gray`.

Base classes: `'rounded px-2 py-0.5 text-xs'`. Add `'border'` when using colorScheme.

Render: `<span className={\`${base} ${colorClasses} ${className}\`} {...props}>{children}</span>`

CRITICAL: The `end` prop MUST be passed through to NavLink. Without it, the "/" route stays active on all pages.
