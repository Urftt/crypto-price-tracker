**Step 1: Update `frontend/src/components/ui/Button.jsx`** (RESP-07 — Button touch targets)

Add mobile touch target minimums to all button sizes and update base classes for vertical centering.

Change the base classes from:
```js
const base = 'rounded cursor-pointer transition-colors disabled:opacity-50 disabled:cursor-not-allowed';
```
to:
```js
const base = 'rounded cursor-pointer transition-colors disabled:opacity-50 disabled:cursor-not-allowed inline-flex items-center justify-center';
```

The `inline-flex items-center justify-center` ensures text stays vertically centered when `min-h-11` is taller than the text + padding.

Change the size map from:
```js
const sizes = {
  xs: 'px-1 py-0.5 text-xs',
  sm: 'px-2.5 py-0.5 text-xs',
  md: 'px-4 py-1.5 text-sm',
};
```
to:
```js
const sizes = {
  xs: 'px-1 py-0.5 text-xs min-h-11 min-w-11 md:min-h-0 md:min-w-0',
  sm: 'px-2.5 py-0.5 text-xs min-h-11 md:min-h-0',
  md: 'px-4 py-1.5 text-sm min-h-11 md:min-h-0',
};
```

`min-h-11` = Tailwind's 2.75rem = 44px. On mobile, all buttons are at least 44px tall. On md+ screens (≥768px), `md:min-h-0` removes the minimum so buttons return to their natural padding-based height.

The `xs` size also gets `min-w-11 md:min-w-0` since tiny icon-like buttons (e.g., the star toggle, close buttons) need width enforcement too.
