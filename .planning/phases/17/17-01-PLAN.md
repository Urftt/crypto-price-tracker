# Phase 17 Plan 01: Mobile Card Layouts

## Goal

Transform data-heavy tables (PriceTable, PortfolioTable, WatchlistPage) into scannable card layouts on mobile (< 640px) using CSS-only `sm:hidden` / `hidden sm:block` toggling, and audit AlertList card spacing for consistency.

## Tasks

### Task 1: Add card layouts to PriceTable and PortfolioTable
**Requirements:** MOB-01, MOB-02

#### Changes

**`frontend/src/components/PriceTable.jsx`** (MOB-01)

Restructure the component to render both a mobile card view and the existing desktop table view. The card view is visible below 640px (`sm:hidden`), the table view is hidden below 640px (`hidden sm:block`).

1. Wrap the existing `<div className="overflow-x-auto">` and its `<Table>` child in a new wrapper div with `className="hidden sm:block"`. This hides the table on mobile.

2. Above the hidden-table wrapper, add the mobile card layout:

```jsx
{/* Mobile card layout */}
<div className="sm:hidden space-y-2">
  {coins.map((coin) => {
    const isWatched = watchlistSymbols.has(coin.symbol);
    return (
      <div
        key={coin.symbol}
        onClick={() => onSelectCoin(coin)}
        className="bg-card border border-border rounded p-3 cursor-pointer active:bg-border/50"
      >
        <div className="flex justify-between items-start mb-1">
          <div>
            <span className="text-accent font-bold">{coin.symbol}</span>
            <span className="text-text-muted text-sm ml-2">{coin.name}</span>
          </div>
          <Button
            variant="ghost"
            size="xs"
            onClick={(e) => toggleWatchlist(e, coin.symbol)}
            type="button"
            className={`text-sm ${isWatched ? 'text-yellow-400' : 'text-text-dim hover:text-yellow-400/60'}`}
            title={isWatched ? 'Remove from watchlist' : 'Add to watchlist'}
          >
            {isWatched ? '\u2605' : '\u2606'}
          </Button>
        </div>
        <div className="flex justify-between items-baseline">
          <span className="text-text">{formatEUR(coin.price)}</span>
          <span className={coin.change_24h > 0 ? 'text-up' : coin.change_24h < 0 ? 'text-down' : 'text-text-muted'}>
            {formatPct(coin.change_24h)}
          </span>
        </div>
      </div>
    );
  })}
</div>
```

Key details:
- Card shows: symbol (bold accent), name (muted, inline), price, 24h% (colored)
- Star button positioned top-right via `flex justify-between items-start`
- Star button uses **the same** `onClick={(e) => toggleWatchlist(e, coin.symbol)}` with `e.stopPropagation()` inside `toggleWatchlist` -- event propagation is preserved
- Card onClick calls `onSelectCoin(coin)` same as table row
- `active:bg-border/50` provides touch feedback
- `cursor-pointer` for visual affordance
- Omits rank (#) and volume columns -- not useful on mobile
- `space-y-2` for consistent 8px gap between cards

The component return becomes:
```jsx
return (
  <>
    {/* Mobile cards */}
    <div className="sm:hidden space-y-2">
      {/* card map */}
    </div>
    {/* Desktop table */}
    <div className="hidden sm:block overflow-x-auto">
      <Table className="max-w-4xl">
        {/* existing thead + tbody unchanged */}
      </Table>
    </div>
  </>
);
```

Move the `overflow-x-auto` class from the current outer div onto the new `hidden sm:block` wrapper div. The Fragment (`<>`) replaces the old outer div.

**`frontend/src/components/PortfolioTable.jsx`** (MOB-02)

Same pattern: add mobile card view above existing table, wrap table in `hidden sm:block`.

1. The early return for empty state (`if (!rows || rows.length === 0)`) stays as-is -- plan 02 will replace it with an EmptyState component.

2. Inside the main return, wrap the existing `<div className="max-w-4xl overflow-x-auto">` with `<Table>` in a new div with `className="hidden sm:block max-w-4xl overflow-x-auto"`. Move `max-w-4xl overflow-x-auto` from the current wrapper onto this new wrapper.

3. Above the hidden-table wrapper, add the mobile card layout:

```jsx
{/* Mobile card layout */}
<div className="sm:hidden space-y-2">
  {rows.map((row) => (
    <div key={row.symbol} className="bg-card border border-border rounded overflow-hidden">
      <div
        onClick={() => toggleLots(row.symbol)}
        className="p-3 cursor-pointer active:bg-border/50"
      >
        <div className="flex justify-between items-start mb-1">
          <div>
            <span className="text-accent font-bold">{row.symbol}</span>
            <span className="text-text-dim text-xs ml-1">
              ({row.num_lots} lot{row.num_lots !== 1 ? 's' : ''})
            </span>
          </div>
          <span className={`text-sm ${row.pnl_pct != null ? pnlColor(row.pnl_pct) : 'text-text-dim'}`}>
            {row.pnl_pct != null ? formatPct(row.pnl_pct) : 'N/A'}
          </span>
        </div>
        <div className="flex justify-between text-sm">
          <span className="text-text-muted">
            Value: {row.current_value != null ? formatEUR(row.current_value) : 'N/A'}
          </span>
          <span className={row.pnl_eur != null ? pnlColor(row.pnl_eur) : 'text-text-dim'}>
            {row.pnl_eur != null ? formatEUR(row.pnl_eur) : 'N/A'}
          </span>
        </div>
      </div>

      {/* Expanded lots section */}
      {expandedSymbol === row.symbol && (
        lotsLoading ? (
          <div className="border-t border-border/50 bg-bg/50 px-3 py-2 text-text-dim text-xs">
            Loading lots...
          </div>
        ) : (
          <div className="border-t border-border/50 bg-bg/50">
            {lots.map((lot) => (
              <div key={`lot-${lot.id}`} className="px-3 py-2 border-b border-border/30 last:border-b-0">
                <div className="flex justify-between items-center text-xs mb-1">
                  <span className="text-text-dim">Lot #{lot.id}</span>
                  <span className="text-text-muted">{formatEUR(lot.buy_price)}</span>
                </div>
                <div className="flex justify-between items-center text-xs">
                  <span className="text-text-muted">
                    {editingLotId === lot.id ? (
                      <input
                        type="number"
                        step="any"
                        value={editAmount}
                        onChange={(e) => setEditAmount(e.target.value)}
                        onKeyDown={(e) => {
                          if (e.key === 'Enter') handleEditLot(lot.id);
                          if (e.key === 'Escape') { setEditingLotId(null); setEditAmount(''); }
                        }}
                        onClick={(e) => e.stopPropagation()}
                        aria-label="Edit amount"
                        className="bg-bg border border-border rounded px-1 py-0.5 text-text text-xs w-20 focus:border-accent focus:outline-none"
                        autoFocus
                      />
                    ) : (
                      <span>Amt: {lot.amount}</span>
                    )}
                  </span>
                  <span className="text-text-dim">{lot.buy_date || '--'}</span>
                </div>
                <div className="flex gap-2 justify-end mt-1">
                  {editingLotId === lot.id ? (
                    <>
                      <Button variant="ghost" size="sm" className="text-up hover:text-up/80" onClick={(e) => { e.stopPropagation(); handleEditLot(lot.id); }} type="button">
                        Save
                      </Button>
                      <Button variant="ghost" size="sm" onClick={(e) => { e.stopPropagation(); setEditingLotId(null); setEditAmount(''); }} type="button">
                        Cancel
                      </Button>
                    </>
                  ) : (
                    <>
                      <Button variant="ghost" size="sm" className="text-accent hover:text-accent/80" onClick={(e) => { e.stopPropagation(); setEditingLotId(lot.id); setEditAmount(String(lot.amount)); }} type="button">
                        Edit
                      </Button>
                      <Button variant="danger" size="sm" onClick={(e) => { e.stopPropagation(); handleDeleteLot(lot.id); }} type="button">
                        Delete
                      </Button>
                    </>
                  )}
                </div>
              </div>
            ))}
          </div>
        )
      )}
    </div>
  ))}
</div>
```

Key details:
- Card header: symbol (bold accent) + lot count (dim), P&L % (colored, top-right)
- Card body: value (muted) and P&L EUR (colored)
- Card tap calls `toggleLots(row.symbol)` -- same state as table rows
- Expanded lots section appears inside card below a `border-t border-border/50` divider
- Lot details show: lot ID, buy price, amount (or edit input), buy date
- Edit/Delete/Save/Cancel buttons use the same `e.stopPropagation()` pattern as the table
- Inline edit input is identical to the table version (same className, aria-label, event handlers)
- `overflow-hidden` on card container clips the rounded corners on expanded section
- `last:border-b-0` prevents double border on last lot
- Null checks on `pnl_eur`, `pnl_pct`, `current_value` matching existing table logic

The component return becomes:
```jsx
return (
  <>
    {/* Mobile cards */}
    <div className="sm:hidden space-y-2">
      {/* card map */}
    </div>
    {/* Desktop table */}
    <div className="hidden sm:block max-w-4xl overflow-x-auto">
      <Table>
        {/* existing thead + tbody unchanged */}
      </Table>
    </div>
  </>
);
```

Replace the current outer `<div className="max-w-4xl overflow-x-auto">` with a Fragment. Move those classes onto the `hidden sm:block` wrapper.

### Task 2: Add card layout to WatchlistPage and audit AlertList spacing
**Requirements:** MOB-03, MOB-04

#### Changes

**`frontend/src/pages/WatchlistPage.jsx`** (MOB-03)

Add a mobile card view for the watchlist data table, using the same `sm:hidden` / `hidden sm:block` pattern.

1. Find the block that renders the filtered entries table (the `else` branch of the `filteredEntries.length === 0` ternary, around line 209). Wrap the existing `<div className="overflow-x-auto"><Table className="max-w-4xl">...</Table></div>` in `<div className="hidden sm:block overflow-x-auto">`. Move `overflow-x-auto` to the new wrapper.

2. Above the hidden-table wrapper (but still inside the else branch), add the mobile card layout:

```jsx
{/* Mobile card layout */}
<div className="sm:hidden space-y-2">
  {filteredEntries.map((entry) => (
    <div
      key={entry.symbol}
      className="bg-card border border-border rounded p-3"
    >
      {/* Header: symbol + name */}
      <div className="flex justify-between items-start mb-2">
        <div>
          <span className="text-accent font-bold">{entry.symbol}</span>
          <span className="text-text-muted text-sm ml-2">{entry.name || '-'}</span>
        </div>
      </div>

      {/* Tags */}
      <div className="mb-2">
        {editingSymbol === entry.symbol ? (
          <div className="flex gap-1 flex-wrap items-center">
            {allTags.map((tag) => (
              <button
                key={tag}
                type="button"
                onClick={() => handleEditTagToggle(tag)}
                className={`px-1.5 py-0.5 rounded text-xs border cursor-pointer transition-opacity min-h-11 md:min-h-0 ${
                  TAG_COLORS[tag] || 'bg-border text-text-muted border-border'
                } ${editTags.includes(tag) ? 'opacity-100 ring-1 ring-accent' : 'opacity-40'}`}
              >
                {tag}
              </button>
            ))}
            <Button variant="ghost" size="sm" className="text-up hover:text-up/80 ml-1" onClick={() => handleSaveTags(entry.symbol)} type="button">
              Save
            </Button>
            <Button variant="ghost" size="sm" onClick={() => { setEditingSymbol(null); setEditTags([]); }} type="button">
              Cancel
            </Button>
          </div>
        ) : (
          <div className="flex gap-1 flex-wrap">
            {entry.tags
              ? entry.tags.split(',').map((tag) => (
                  <Badge key={tag} colorScheme={TAG_COLOR_MAP[tag] || 'gray'} className="px-1.5 py-0.5">
                    {tag}
                  </Badge>
                ))
              : <span className="text-text-dim text-xs">No tags</span>}
          </div>
        )}
      </div>

      {/* Price + change */}
      <div className="flex justify-between items-baseline mb-2">
        <span className="text-text text-sm">
          {entry.price != null ? formatEUR(entry.price) : <span className="text-text-dim">N/A</span>}
        </span>
        <span className={`text-sm ${
          entry.change_24h > 0 ? 'text-up' : entry.change_24h < 0 ? 'text-down' : 'text-text-muted'
        }`}>
          {entry.change_24h != null ? formatPct(entry.change_24h) : <span className="text-text-dim">N/A</span>}
        </span>
      </div>

      {/* Actions */}
      <div className="flex gap-2 pt-1 border-t border-border/30">
        <Button
          variant="ghost"
          size="sm"
          className="text-accent hover:text-accent/80"
          onClick={() => handleStartEditTags(entry)}
          type="button"
          title={`Edit tags for ${entry.symbol}`}
        >
          Edit Tags
        </Button>
        <Button
          variant="danger"
          size="sm"
          onClick={() => handleRemove(entry.symbol)}
          type="button"
          title={`Remove ${entry.symbol} from watchlist`}
        >
          Remove
        </Button>
      </div>
    </div>
  ))}
</div>
```

Key details:
- Card layout: symbol (bold accent) + name (muted), tag Badge pills, price + 24h% change, action buttons
- Tag pills use the same `Badge` component with `TAG_COLOR_MAP` colorScheme -- identical to the table column
- Inline tag editing works identically: toggle buttons for each tag with Save/Cancel, same `handleEditTagToggle`, `handleSaveTags`, same classNames for tag buttons
- Tag toggle buttons keep `min-h-11 md:min-h-0` from Phase 16 touch target work
- Action buttons (Edit Tags, Remove) separated by `border-t border-border/30` divider
- Volume column omitted on mobile -- less important for watchlist scanning
- No `cursor-pointer` or `active:bg-border/50` on card itself -- watchlist cards are not clickable (no coin modal), only the buttons are interactive
- The empty state rendering for `filteredEntries.length === 0` stays as-is -- plan 02 replaces it
