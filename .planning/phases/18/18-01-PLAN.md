## Task 3: Modal Enter/Exit Animations (VIS-02)

**File(s):** `frontend/src/components/ui/Modal.jsx`
**What:** Replace instant show/hide with animated enter/exit. Use a two-state approach (open = desired state, visible = DOM presence) with CSS transitions for both the backdrop (opacity) and the card (scale + opacity). Exit animation plays before DOM unmount.
**Depends on:** Task 1

### Changes

Replace the entire `Modal.jsx` with the following implementation. The key changes are:
- Add `useState` to imports (alongside existing `useEffect`, `useRef`)
- Add `visible` state to control DOM presence
- Replace `if (!open) return null` with the two-state approach
- Add CSS transition classes to backdrop and card
- Use `onTransitionEnd` on the backdrop to detect when exit animation completes

```jsx
import { useState, useEffect, useRef } from 'react';

export function Modal({ open, onClose, children }) {
  const modalRef = useRef(null);
  const [visible, setVisible] = useState(false);

  // Sync visible state with open prop
  useEffect(() => {
    if (open) {
      setVisible(true);
    }
  }, [open]);

  // When exit transition ends, remove from DOM
  const handleTransitionEnd = (e) => {
    // Only respond to the backdrop's own transition, not bubbled events
    if (e.target === e.currentTarget && !open) {
      setVisible(false);
    }
  };

  // Escape key handler
  useEffect(() => {
    if (!open) return;

    function handleKeyDown(e) {
      if (e.key === 'Escape') {
        onClose();
      }
    }

    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
  }, [open, onClose]);

  // Focus trap
  useEffect(() => {
    if (!open || !modalRef.current) return;

    const focusableSelector =
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])';

    const focusable = modalRef.current.querySelectorAll(focusableSelector);
    if (focusable.length > 0) {
      focusable[0].focus();
    }

    function handleTab(e) {
      if (e.key !== 'Tab') return;

      const currentFocusable = modalRef.current.querySelectorAll(focusableSelector);
      if (currentFocusable.length === 0) return;

      const first = currentFocusable[0];
      const last = currentFocusable[currentFocusable.length - 1];

      if (e.shiftKey) {
        if (document.activeElement === first) {
          e.preventDefault();
          last.focus();
        }
      } else {
        if (document.activeElement === last) {
          e.preventDefault();
          first.focus();
        }
      }
    }

    document.addEventListener('keydown', handleTab);
    return () => document.removeEventListener('keydown', handleTab);
  }, [open]);

  if (!visible) return null;

  return (
    <div
      className={`fixed inset-0 z-40 flex items-center justify-center transition-opacity duration-200 ${
        open ? 'bg-black/70 opacity-100' : 'bg-black/70 opacity-0'
      }`}
      onClick={onClose}
      onTransitionEnd={handleTransitionEnd}
    >
      <div
        ref={modalRef}
        className={`bg-card border-border-light w-full h-full sm:border sm:rounded-lg sm:p-6 sm:max-w-xl sm:h-auto sm:mx-4 p-4 relative z-50 overflow-y-auto transition-all duration-200 ${
          open ? 'scale-100 opacity-100' : 'scale-95 opacity-0'
        }`}
        onClick={(e) => e.stopPropagation()}
      >
        {children}
      </div>
    </div>
  );
}
```

Key details:
- When `open` becomes true: `visible` is set to true (mounts DOM), then CSS classes apply `opacity-100` and `scale-100`
- When `open` becomes false: CSS classes switch to `opacity-0` and `scale-95`, the 200ms transition plays, then `onTransitionEnd` sets `visible` to false (unmounts DOM)
- The `onTransitionEnd` handler checks `e.target === e.currentTarget` to avoid responding to child element transitions
- Escape key handler and focus trap remain unchanged
- The card uses `transition-all duration-200` which covers both `opacity` and `transform` (scale) properties

## Task 4: Toast Enter/Exit Animations (VIS-03)

**File(s):** `frontend/src/components/Toast.jsx`
**What:** Add slide-in-right animation for enter and fade-out animation for exit. Introduce a `dismissing` state so the toast stays in the DOM during its exit animation, with `onAnimationEnd` triggering actual removal.
**Depends on:** Task 1

### Changes

Replace the entire `Toast.jsx` with the following. Key changes:
- Add `useState` to the import
- Add `dismissing` state
- Auto-dismiss timer sets `dismissing: true` instead of calling `onClose` directly
- Close button sets `dismissing: true` instead of calling `onClose` directly
- `onAnimationEnd` calls `onClose` when in dismissing state (removes from DOM)
- Dynamic class: `animate-slide-in-right` when entering, `animate-fade-out` when dismissing

```jsx
import { useState, useEffect } from 'react';
import { Button } from './ui/Button';

function Toast({ message, type = 'info', duration = 10000, onClose, style }) {
  const [dismissing, setDismissing] = useState(false);

  useEffect(() => {
    const timer = setTimeout(() => setDismissing(true), duration);
    return () => clearTimeout(timer);
  }, [duration]);

  const handleAnimationEnd = () => {
    if (dismissing) onClose();
  };

  const colors = {
    info: 'border-accent text-accent',
    success: 'border-up text-up',
    error: 'border-down text-down',
  };

  return (
    <div
      className={`fixed right-4 z-50 bg-card border ${colors[type]} rounded px-4 py-3 font-mono text-sm shadow-lg ${
        dismissing ? 'animate-fade-out' : 'animate-slide-in-right'
      }`}
      style={style}
      onAnimationEnd={handleAnimationEnd}
    >
      {message}
      <Button variant="ghost" size="xs" onClick={() => setDismissing(true)} type="button" className="ml-3">
        x
      </Button>
    </div>
  );
}

export default Toast;
```

Key details:
- `onClose` is deliberately removed from the `useEffect` dependency array -- the timer should NOT reset if `onClose` changes identity. Only `duration` matters for the auto-dismiss timer.
- `animate-fade-out` uses `forwards` fill mode (defined in index.css) so the toast stays invisible after the animation ends, preventing a flash-back before `onAnimationEnd` fires and removes the element.
- The close button (`x`) now calls `setDismissing(true)` instead of `onClose()` directly, so the exit animation plays before removal.
- The enter animation (`animate-slide-in-right`) plays once on mount and does not interfere with the exit animation because the class switches to `animate-fade-out` when `dismissing` becomes true.

**Depends on:** Task 1


2. **Add price tracking state and refs** inside the `PriceTable` function, after the `const [watchlistSymbols, setWatchlistSymbols]` line:

3. **Add a useEffect for price change detection**, after the existing `useEffect` that calls `loadWatchlist()`:

   ```jsx
   useEffect(() => {
     if (isFirstRender.current) {
       // First render: record prices but don't flash
       for (const coin of coins) {
         prevPrices.current[coin.symbol] = coin.price;
       }
       isFirstRender.current = false;
       return;
     }

     const flashing = new Set();
     for (const coin of coins) {
       const prev = prevPrices.current[coin.symbol];
       if (prev !== undefined && prev !== coin.price) {
         flashing.add(coin.symbol);
       }
       prevPrices.current[coin.symbol] = coin.price;
     }
     if (flashing.size > 0) {
       setFlashSymbols(flashing);
       const timer = setTimeout(() => setFlashSymbols(new Set()), 600);
       return () => clearTimeout(timer);
     }
   }, [coins]);
   ```

4. **Apply flash class to mobile cards.** In the mobile card layout, change the card div's className from:
   ```jsx
   className="bg-card border border-border rounded p-3 cursor-pointer active:bg-border/50"
   ```
   to:
   ```jsx
   className={`bg-card border border-border rounded p-3 cursor-pointer active:bg-border/50 ${flashSymbols.has(coin.symbol) ? 'animate-flash' : ''}`}
   ```
   Note: this changes from a string literal to a template literal.

5. **Apply flash class to desktop table rows.** In the desktop table layout, change the `<tr>` className from:
   ```jsx
   className="cursor-pointer hover:bg-card border-b border-border/50"
   ```
   to:
   ```jsx
   className={`cursor-pointer hover:bg-card border-b border-border/50 ${flashSymbols.has(coin.symbol) ? 'animate-flash' : ''}`}
   ```
   Note: this changes from a string literal to a template literal.

Key details:
- `isFirstRender` ref ensures no flash on initial page load (when all prices are "new")
- `prevPrices` ref persists across renders without causing re-renders
- The 600ms timeout matches the `animate-flash` keyframe duration defined in index.css
- `flashSymbols` is a Set for O(1) lookup per row
- The flash applies to both mobile cards and desktop table rows
- When a coin is in `flashSymbols`, the `animate-flash` keyframe runs once (600ms), then the timeout clears the set, removing the class
- The `animate-flash` keyframe briefly highlights with 25% opacity accent color at the 15% mark, then fades back to transparent
